Set env for Clojure with IntelliJ and Leiningen

投稿日 2015-09-07   |  

起步Clojure编程．..

OS: Ubuntu 14.10

IDEA 14.0.3 Ultimate

安装Leiningen
按照 http://leiningen.org 的指南安装好lein

安装La Clojure
安装Idea插件La Clojure。启动Idea，点左上角的File –> Settings –> Plugins –> 搜”Clojure” , 然后找到La Clojure, 然后Install。

新建项目
在workspace下建立clojure工程。输入模式: lein new app groupId/artifactId。groupId和artifactId和Maven里的概念一致

实例: $lein new app cn.kenshinn/clojure_demo， 就会依照模版建立一个clojure工程。

导入IDEA
这个工程直接导入IDEA不会被正确识别，因此需要在clojure_demo目录下执行lein pom，生成相应的pom。

然后在IDEA中import project,　选择工程目录, 然后点Maven，一路点下去。

配置main函数
在打开的IDEA工程中，打开src目录，在cn.kenshinn包中有clojure_demo.clj文件。

这个clojure文件中，并没有main函数，因此在IDEA中执行run, 什么也不会输出。

把这个文件的内容改一下

然后在IDEA中最上边的菜单中点Run， 选择Edit Configurations, 选中Run main function in the script namespace，点OK。

在Run菜单中执行Run “clojure_demo”， 程序会打印出”Hello, World!”。

这时候，如果cmd下执行lein run， 会说“No :main namespace specified in project.clj”.

需要修改project.clj加上:main cn.kenshinn.clojure_sample指定main函数

保存后，执行lein run，输出”Hello, world!”

Perfect !!!
