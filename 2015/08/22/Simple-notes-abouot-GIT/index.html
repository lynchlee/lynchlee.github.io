  22. 多人协作的工作模式通常是这样
    1). 首先，可以试图用git push origin branch-name推送自己的修改； 2). 如果推送失败，则因为远程分支比你的本地更新，需要先用git pull试图合并； 3). 如果合并有冲突，则解决冲突，并在本地提交； 4). 没有冲突或者解决掉冲突后，再用git push origin branch-name推送就能成功！ 如果git pull提示“no tracking information”，则说明本地分支和远程分支的链接关系没有创建，用命令 git branch --set-upstream branch-name origin/branch-name
